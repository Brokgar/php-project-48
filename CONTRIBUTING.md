# Руководство для контрибьюторов

Благодарим за интерес к проекту Gendiff! Этот документ содержит рекомендации по внесению изменений в проект.

## Как внести свой вклад

### 1. Форк и клонирование

1. Создайте форк репозитория на GitHub
2. Клонируйте форк на локальную машину:
   ```bash
   git clone https://github.com/YOUR_USERNAME/php-project-48.git
   cd php-project-48
   ```

### 2. Установка зависимостей

```bash
make install
```

### 3. Создание ветки

Создайте новую ветку для вашей функции или исправления:

```bash
git checkout -b feature/your-feature-name
# или
git checkout -b fix/your-bug-fix
```

### 4. Разработка

#### Требования к коду

- Код должен соответствовать стандарту **PSR-12**
- Все публичные методы должны иметь **PHPDoc** блоки
- Код должен проходить статический анализ **PHPStan** (уровень 5)

#### Проверка кода

Перед коммитом убедитесь, что код проходит все проверки:

```bash
# Проверка стиля кода
make lint

# Статический анализ
make phpstan

# Запуск тестов
make test

# Все проверки сразу
make analyze
```

#### Написание тестов

- Новые функции должны сопровождаться тестами
- Покрытие кода не должно уменьшаться
- Тесты должны быть изолированными и воспроизводимыми

### 5. Коммиты

- Используйте понятные сообщения коммитов
- Следуйте соглашениям [Conventional Commits](https://www.conventionalcommits.org/)

Примеры:
```
feat: добавить поддержку формата YAML на вывод
fix: исправить обработку вложенных структур в StylishFormatter
docs: обновить README.md
test: добавить тесты для ArrayComparator
```

### 6. Pull Request

1. Запушьте изменения в ваш форк
2. Создайте Pull Request в основной репозиторий
3. Опишите изменения в описании PR
4. Дождитесь ревью и CI-проверок

## Структура проекта

```
php-project-48/
├── src/                    # Исходный код библиотеки
│   ├── Exception/          # Исключения
│   ├── Formatters/         # Форматтеры вывода
│   ├── Utils/              # Вспомогательные классы
│   └── ...
├── tests/                  # Тесты
│   └── fixtures/           # Тестовые данные
├── bin/                    # Исполняемые файлы
└── build/                  # Отчёты (генерируются автоматически)
```

## Запуск утилиты

```bash
# Простое сравнение
php bin/gendiff tests/fixtures/file1.json tests/fixtures/file2.json

# С указанием формата
php bin/gendiff -f stylish tests/fixtures/file1.json tests/fixtures/file2.json

# Показать справку
php bin/gendiff --help
```
